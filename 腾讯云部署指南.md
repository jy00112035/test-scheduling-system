## 2. 服务器基础配置

### 2.1 SSH 登录服务器

**Mac / Linux：**
```bash
ssh root@你的服务器公网IP
```

**Windows（用 PowerShell）：**
```powershell
ssh root@你的服务器公网IP
```

输入密码即可登录。

### 2.2 更新系统包

```bash
apt update && apt upgrade -y
```

### 2.3 开放防火墙端口

腾讯云轻量服务器需要在 **控制台** → **防火墙** 中添加规则：

| 端口 | 用途 |
|:---:|:----:|
| 80 | HTTP 访问 |
| 443 | HTTPS 访问 |
| 8080 | 后端 API（内网用，可不对外开放）|
| 3306 | MySQL（内网用，**不要对外**）|

> **安全建议**：只需开放 80 和 443 端口给公网，8080 和 3306 通过内网或 Nginx 反向代理访问。

## 3. 安装 MySQL

### 3.1 安装 MySQL 8.0

```bash
apt install mysql-server -y
```

### 3.2 初始化安全配置

```bash
mysql_secure_installation
```

按提示操作：
- 输入 root 密码（设置一个，如 `TestScheduling2024!`）
- 移除匿名用户 → `Y`
- 禁止 root 远程登录 → **`N`**（保留本机登录即可，应用和数据库在同一台服务器）
- 移除 test 数据库 → `Y`
- 重新加载权限表 → `Y`

### 3.3 创建应用数据库和用户

```bash
mysql -u root -p
```

输入刚才设置的 root 密码，进入 MySQL 后执行：

```sql
-- 创建数据库
CREATE DATABASE test_scheduling DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建应用用户（设置密码后请记下来，后面要用）
CREATE USER 'scheduling'@'localhost' IDENTIFIED BY '你的密码';

-- 授权
GRANT ALL PRIVILEGES ON test_scheduling.* TO 'scheduling'@'localhost';
FLUSH PRIVILEGES;

-- 退出
EXIT;
```

### 3.4 验证数据库

```bash
mysql -u scheduling -p -D test_scheduling -e "SELECT 1;"
```

看到 `1` 说明成功。

## 6. 配置后端项目（切换 MySQL）

### 6.1 上传项目到服务器

**方式一：从你 Mac 上直接 scp 上传**

在你自己电脑上（不是服务器）执行：
```bash
# 打包项目（排除 node_modules）
cd ~/Desktop/code/ceshi-v1.1
tar --exclude=node_modules -czf ceshi-v1.1.tar.gz .

# 上传到服务器
scp ceshi-v1.1.tar.gz root@你的服务器IP:/root/
```

然后在服务器上解压：
```bash
mkdir -p /opt/test-scheduling
tar -xzf /root/ceshi-v1.1.tar.gz -C /opt/test-scheduling
cd /opt/test-scheduling
```

**方式二：用 Git（如果有）**
```bash
# 在你电脑上推送到 GitHub/Gitee，然后在服务器上 clone
cd /opt
git clone 你的仓库地址 test-scheduling
```

### 6.2 修改后端数据库配置

编辑 `backend/src/main/resources/application.yml`：

```yaml
server:
  port: 8080

spring:
  application:
    name: test-scheduling-backend

  datasource:
    url: jdbc:mysql://localhost:3306/test_scheduling?useUnicode=true&characterEncoding=utf8mb4&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true
    driver-class-name: com.mysql.cj.jdbc.Driver
    username: scheduling
    password: 你刚才设置的密码

  # 以下删掉或注释掉 H2 相关配置
  # h2:
  #   console:
  #     enabled: true
  #     path: /h2-console

  jpa:
    hibernate:
      ddl-auto: update
    show-sql: false
    properties:
      hibernate:
        format_sql: true
        dialect: org.hibernate.dialect.MySQLDialect
    # ↓ 新增：关闭 Open-in-View 避免懒加载问题（可选）
    open-in-view: false

  jackson:
    time-zone: GMT+8
    date-format: yyyy-MM-dd HH:mm:ss
    serialization:
      write-dates-as-timestamps: false

jwt:
  secret: TestSchedulingSecretKey2024VeryLongSecretKeyForJWTTokenGeneration
  expiration: 86400000

logging:
  level:
    com.testscheduling: INFO
    org.hibernate.SQL: WARN  # 生产环境减少日志

```

> ⚠️ **重点**：将 `password` 替换为你在第 3.3 步创建的 MySQL 用户密码。

### 6.3 修改前端 API 地址（生产构建用）

编辑 `src/services/api.ts`，确认第一行是（之前已修改过）：

```typescript
const API_BASE_URL = `${window.location.protocol}//${window.location.hostname}:8080/api`;
```

> 生产环境走 Nginx 反向代理后，可以改为相对路径：
> ```typescript
> const API_BASE_URL = '/api';
> ```
> 这样前端请求 `/api/xxx` 由 Nginx 转发到后端 `localhost:8080/api/xxx`。

## 8. 构建并启动后端

### 8.1 构建后端 JAR 包

```bash
cd /opt/test-scheduling/backend
mvn clean package -DskipTests -q
```

构建完成后，JAR 包在 `target/` 目录下：
```bash
ls target/*.jar
# 应该看到 test-scheduling-backend-1.0.0.jar
```

### 8.2 配置系统服务（开机自启）

创建 systemd 服务文件：

```bash
vim /etc/systemd/system/test-scheduling-backend.service
```

粘贴以下内容：

```ini
[Unit]
Description=测试排班系统后端
After=network.target mysql.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/test-scheduling/backend
ExecStart=/usr/bin/java -jar /opt/test-scheduling/backend/target/test-scheduling-backend-1.0.0.jar
Restart=on-failure
RestartSec=10
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

### 8.3 启动后端

```bash
# 重新加载 systemd
systemctl daemon-reload

# 启动服务
systemctl start test-scheduling-backend

# 设置开机自启
systemctl enable test-scheduling-backend

# 查看状态
systemctl status test-scheduling-backend

# 查看实时日志
journalctl -u test-scheduling-backend -f
```

看到 `Started TestSchedulingApplication` 即为启动成功。

验证 API：
```bash
curl -s http://localhost:8080/api/auth/login -X POST -H "Content-Type: application/json" -d '{"username":"admin","password":"admin123"}'
```

应返回包含 `token` 的 JSON。

## 9. 配置 Nginx 反向代理

### 9.1 安装 Nginx

```bash
apt install nginx -y
```

### 9.2 配置 Nginx

```bash
vim /etc/nginx/sites-available/test-scheduling
```

粘贴以下内容（**将域名替换成你自己的**）：

```nginx
# HTTP -> HTTPS 重定向（等配好 SSL 证书后生效）
server {
    listen 80;
    server_name 你的域名.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name 你的域名.com;

    # SSL 证书（见第 10 节）
    # ssl_certificate     /etc/ssl/certs/你的域名.pem;
    # ssl_certificate_key /etc/ssl/private/你的域名.key;

    # 前端静态文件
    root /opt/test-scheduling/dist;
    index index.html;

    # 前端路由（支持 React Router 的 history 模式）
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 大请求体支持（导入数据可能需要）
        client_max_body_size 50m;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

> **注意**：`# ssl_certificate` 那两行先注释掉，等第 10 节配好 SSL 证书后再取消注释。

### 9.3 启用配置

```bash
# 删除默认站点
rm /etc/nginx/sites-enabled/default

# 启用我们的配置
ln -sf /etc/nginx/sites-available/test-scheduling /etc/nginx/sites-enabled/

# 测试配置
nginx -t

# 重新加载 Nginx
systemctl reload nginx
```

### 9.4 同时修改前端 API 地址为相对路径

因为现在走 Nginx 反向代理了，API 请求改回相对路径更简洁：

编辑 `src/services/api.ts`：
```typescript
const API_BASE_URL = '/api';
```

然后重新构建前端：
```bash
cd /opt/test-scheduling
npm run build
```

## 10. 配置域名与 HTTPS（SSL 证书）

### 10.1 域名 DNS 解析

在域名服务商（如腾讯云 DNSPod）的控制台，添加一条 **A 记录**：

| 记录类型 | 主机记录 | 记录值 | TTL |
|:-------:|:-------:|:------:|:---:|
| A | @ | 你的服务器公网 IP | 600 |
| A | www | 你的服务器公网 IP | 600 |

> DNS 生效可能需要几分钟到几小时。

### 10.2 申请 SSL 证书

**方式一：Certbot（Let's Encrypt，免费推荐）**

```bash
# 安装 Certbot
apt install certbot python3-certbot-nginx -y

# 申请证书（会自动修改 Nginx 配置）
certbot --nginx -d 你的域名.com -d www.你的域名.com
```

按提示输入邮箱，同意协议即可。证书会自动续期。

**方式二：腾讯云免费 SSL 证书**

1. 登录 [腾讯云 SSL 证书控制台](https://console.cloud.tencent.com/ssl)
2. 点击「申请免费证书」→ 选择「域名型免费证书（DV）」
3. 填写你的域名，DNS 验证
4. 下载证书，选择 **Nginx** 格式
5. 上传到服务器：
   ```bash
   # 在服务器上创建目录
   mkdir -p /etc/ssl/certs /etc/ssl/private
   ```
6. 用 scp 从你电脑上传证书文件到服务器，然后修改 Nginx 配置取消 ssl 行注释

### 10.3 验证 HTTPS

```bash
# 执行后，certbot 已自动修改了 Nginx 配置
systemctl reload nginx
```

浏览器访问 `https://你的域名.com`，应该能看到小锁标志 ✅

## 13. 附：常用运维命令

### 服务管理

```bash
# 后端
systemctl start|stop|restart|status test-scheduling-backend
journalctl -u test-scheduling-backend -f     # 查看实时日志

# Nginx
systemctl reload|restart|status nginx
nginx -t                                      # 测试配置

# MySQL
systemctl start|stop|restart|status mysql
```

### 更新部署流程（下次改代码后）

```bash
# 1. 停服务
systemctl stop test-scheduling-backend

# 2. 拉取新代码 / 上传新文件
cd /opt/test-scheduling
# scp 或 git pull ...

# 3. 重新构建后端
cd /opt/test-scheduling/backend
mvn clean package -DskipTests -q

# 4. 重新构建前端（如果前端有改动）
cd /opt/test-scheduling
npm install
npm run build

# 5. 启动服务
systemctl start test-scheduling-backend

# 6. 检查日志
journalctl -u test-scheduling-backend -f --since "1 min ago"
```

### 备份数据库

```bash
# 备份
mysqldump -u scheduling -p test_scheduling > /root/backup/backup_$(date +%Y%m%d).sql

# 恢复
mysql -u scheduling -p test_scheduling < /root/backup/backup_20250101.sql
```

> 💡 建议设置 crontab 定时备份：
> ```bash
> crontab -e
> # 添加：每天凌晨 3 点备份
> 0 3 * * * mysqldump -u scheduling -p'你的密码' test_scheduling > /root/backup/backup_$(date +\%Y\%m\%d).sql
> ```
